# CHANGELOG

本文档用于补录 TindaAgent 的版本演进历史，后续按版本持续维护。

## v1.7.13 - 2026-05-02

1. 启动器增强：端口占用自动重试（最多 20 次顺延）、`--open-browser` 自动打开浏览器、`--port-retries` 可配。
2. 启动脚本完善：start.bat/sh 支持自动探测 Python 路径（py/python/conda/venv）、stop.bat/sh 停止脚本、status.bat/sh/ps1 状态查看。
3. 审计日志根目录回退：WSL/容器只读挂载时自动切换可写目录。
4. 项目架构文档：`docs/architecture.md`，Mermaid 函数级调用链图。
5. 代码格式化统一与多轮 bug 修复。

## v1.7.12 - 2026-05-01

1. 终端执行内联气泡：聊天区直接显示命令、输出、返回码，不再只靠终端面板。
2. 修复 `terminal_policy.py` 漏了 `import json`，导致 bypass 检查永远返回空字典，确认流程形同虚设。
3. 修复审计批量刷盘计时器从未启动，进程关闭前的事件全部丢失。
4. Agent 新增 `refresh_model_identity()`，模型切换或规则变更时保留会话正文。
5. LLM 消息校验 `_require_user_last_message()`，防止空消息或角色错位。

## v1.7.11 - 2026-04-30

因为 AI 辅助写代码引入的 bug 有点多，导致这一版反复修了好几次才稳定。我发誓下次应该更仔细地规划再动手。

1. 新增终端执行工具 `run_terminal`：LLM 可直接执行 shell 命令，权限要求 63，系统操作额外校验 SYSTEM_EXECUTE。
2. 终端安全策略模块：命令白名单/黑名单、22 种系统操作自动检测、bypass 确认模式。
3. 终端确认气泡 + 设置页终端控制面板。
4. 移除假工具调用检测，误报太多。
5. 修了一堆 AI 搞出来的 bug：token 索引属性名、_caller_perm 参数注入、缓存递归爆栈、import os 遗漏。

## v1.7.10 - 2026-04-29

1. 性能优化：审计批量刷盘（16条/3秒缓冲）、JSONL 增量追加、会话元数据攒批写入、消息热缓存（LRU 50会话）。
2. 解耦优化：用户查询三索引 O(1)、假调用检测编译正则单次扫描、工具列表+Schema缓存。
3. Agent base_history 惰性缓存，仅在 prompt 或记忆变化时重建。

## v1.7.9 - 2026-04-29

1. 日志查看器视觉统一：采用终端面板同款粉色渐变背景、macOS 三色点头部、JetBrains Mono 等宽字体、自定义粉色滚动条。
2. 设置页快捷按钮改为操作入口 + 显示开关双栏设计，可直接从设置页跳转日志/模型检测。

## v1.7.8 - 2026-04-29

1. 新增设置页面（`/settings`）：延续可爱风格，集中管理流式输出、终端、token 阈值、快捷按钮等配置项。
2. 聊天页 Header 重构：快捷按钮系统（动态渲染 + 用户自选）、账户切换改为单一按钮 + 弹出卡片窗口、仅保留设置与返回主页为固定按钮。
3. Header 布局简化：默认显示模型切换、流式输出、终端、压缩上下文四个快捷按钮，其余可在设置页开启。

## v1.7.7 - 2026-04-29

1. 新增 token 估算模块：英文 ~4 字符/token，中文 ~1.5 字符/token，每次请求自动估算上下文 token 用量并记录审计日志。
2. 新增自动上下文压缩：超过阈值（默认 16K tokens）时在 LLM 调用前自动触发压缩，复用已有 LLM 摘要 + SessionStore 压缩逻辑。
3. 新增阈值配置 UI：聊天页 header 齿轮按钮，弹窗可自由设置每会话 token 阈值（100 ~ 10M），持久化到会话元数据。
4. 增强伪工具调用检测：改用前缀匹配表，新增否定句式排除（如"不需要调用工具"），减少误报。
5. SessionStore 会话元数据新增 `max_context_tokens` 字段，Agent 支持 `estimate_current_tokens()` 和 `_should_compress()`。

## v1.7.6 - 2026-04-28

1. 新增聊天页文件导入功能：支持通过附件按钮导入文本文件，文件内容自动注入对话，用户消息显示文件芯片气泡，session 持久化支持 `attachment` 条目类型。
2. 审计日志自动轮转：`total.jsonl` 超过 10MB 自动压缩归档为 `.gz`，同时维护 `.idx` 索引文件实现 O(1) 事件 ID 偏移查找。
3. 模型检测页 UI 翻新：接入 Google Fonts（Nunito + JetBrains Mono），更新主题配色与主站统一，新增浮动装饰动画，检测结果改为独立卡片式历史记录（保留最近 10 次），每次运行展示模型/测试项/时间/PASS-UNSUPPORTED-FAIL-SKIPPED 统计。

## v1.7.5 - 2026-04-27

1. 新增独立模型检测页（`/model-diagnostics`），支持单项检测与一键全部检测。
2. 新增模型能力检测 API（`POST /model-diagnostics/run`），覆盖连接测试、思考支持测试、图片 URL 测试、视频 URL 测试。
3. 聊天页新增“模型检测”入口按钮，并按 LLM 权限位（`PUBLIC_EXECUTE`）控制显示。

## v1.7.4 - 2026-04-27

1. 终端性能优化：新增终端行数上限裁剪（最多 500 行）、会话事件批量写入（缓冲合并）、工具事件批量 DOM 渲染，显著降低长会话卡顿。
2. 终端显示优化：报错输出统一高亮，刷新/重载后保持错误颜色。
3. 终端结构优化：同一次工具请求（cmd 到 sep）合并为单个终端气泡，便于阅读与定位。
4. 修复审计调用兼容：`audit_event` 同时兼容位置参数与关键字参数，避免旧调用链触发参数错误。

## v1.7.3 - 2026-04-27

1. 修复用户管理入口权限一致性：聊天页与首页统一按 `USER_ADMIN(511)` 判断显示“用户管理”，并在 `/user-admin` 后端路由增加权限拦截与安全回跳（优先 `next/referer`）。
2. 修复版本状态长期分裂：新增运行时版本自愈对齐逻辑，`/system/version` 增加 `effective_version` 与 `switch_enabled`，检测到历史 `current.json` 偏移时自动对齐运行版本。
3. 增强聊天页版本气泡：点击版本气泡时即时调用 `/system/versions` 检查远端最新版本，明确提示“已是最新/发现新版本/检查失败”。
4. 优化工具事件轮询错误体验：`/sessions/{id}/tool-events` 网络抖动（如 `Failed to fetch`）改为节流与连续失败再提示，避免工具成功后仍高频刷红错。
5. 增强日志 ID 兼容解析：`get_log_event_by_id` 支持 `log#123` / `log-123` / `log:123` 等输入格式，兼容原有 `#123` 与 `tc_` 前缀。

## v1.7.2 - 2026-04-26

2. 优化发送流程清理逻辑：`sendMessage()` 增加统一清理函数，覆盖流式/非流式成功与异常分支，减少分支遗漏。
